# Synapse Database Schema

## Overview

This document describes the database design for Synapse.

The schema is designed to support:

* Debate execution
* Multi-model interactions
* Consensus generation
* Evaluation metrics
* Research experiments

The database follows a relational model using PostgreSQL.

---

# Design Principles

## Store Events, Not Agents

Agents are execution logic.

The database stores outputs generated during debates.

---

## Preserve Debate History

All debate stages are persisted.

This enables:

* Reproducibility
* Behavioral analysis
* Future research

---

## Extensibility

The schema should support:

* New models
* Additional debate rounds
* New evaluation metrics
* Future experiments

without major structural changes.

---

# Table: debates

Stores high-level debate information.

## Columns

| Column       | Type      |
| ------------ | --------- |
| id           | UUID      |
| question     | TEXT      |
| status       | VARCHAR   |
| total_rounds | INTEGER   |
| created_at   | TIMESTAMP |
| completed_at | TIMESTAMP |

---

## Example

```json
{
  "id": "deb_001",
  "question": "Should AI-generated code be deployed directly to production?",
  "status": "completed"
}
```

---

---

| Column     | Type      |

# Table: responses

Stores all model-generated outputs.

This is the most important table in the system.

---

## Columns

| Column           | Type      |
| ---------------- | --------- |
| id               | UUID      |
| debate_id        | UUID      |
| round_id         | UUID      |
| participant_id   | UUID      |
| response_type    | VARCHAR   |
| content          | TEXT      |
| confidence_score | FLOAT     |
| created_at       | TIMESTAMP |

---

## Response Types

```text
opinion
critique
defense
revision
consensus_input
```

---

## Example

```json
{
  "response_type": "critique",
  "content": "This argument assumes perfect human review processes.",
  "confidence_score": 78
}
```

---

# Schema Summary

The Synapse schema is designed around debates as the primary entity. Every interaction, response, metric, and consensus artifact is linked back to a debate, enabling reproducible experiments and long-term analysis of multi-model reasoning behavior.
